# 📘 議事録 運用ガイド

チームで議事録の品質を統一し、「言った言わない」問題を防止するための運用ルールです。

---

## 1. 議事録作成の基本ルール

### 作成タイミング
- **会議当日中**に初版を作成（遅くとも翌営業日まで）
- 記憶が鮮明なうちに作成することが最も重要

### 記載の原則
- **「だ・である調」**で統一（敬語は不要）
- **1文1情報**を原則とし、簡潔に
- **発言者を必ず記録**する（「〇〇氏より△△との発言あり」）
- **決定事項と個人の意見は明確に区別**する
- **客観的な記述**に徹する（主観・推測を入れない）

### 5W2H を意識
議事録の核となる情報は以下を網羅しているか確認：
- **Who**（誰が） - 発言者、決定者、担当者
- **What**（何を） - 決定内容、タスク内容
- **When**（いつまでに） - 期限、スケジュール
- **Where**（どこで） - 実施場所、共有先
- **Why**（なぜ） - 決定の理由、背景
- **How**（どうやって） - 具体的な方法
- **How much**（いくらで） - 予算、コスト

---

## 2. 会議前の準備

### 必ずやること
1. **前回の議事録を確認**し、未完了のTODOを洗い出す
2. **アジェンダを事前に共有**（できれば前日まで）
3. **テンプレートを事前に準備**（出席者・議題を先に記入）
4. **配布資料を事前に共有**（会議中に初見は避ける）

### テンプレートの事前記入チェック
- [ ] 会議名称
- [ ] 日時・場所
- [ ] 出席予定者
- [ ] アジェンダ（議題一覧）
- [ ] 前回TODOの転記

---

## 3. 会議中のメモ術

### 走り書きのコツ
- **`名前>`** で発言者を即記録（たった1文字のルール）
- **完璧な文章を書かない** → キーワードだけで十分
- **決定事項はその場で `★` マーク**を付ける
- **聞き取れなかったら `？？？` と書く** → 会議後に確認
- **録画をONにする**（後から確認できる安心感）

### 走り書き例
```
田中> A案件は3月末で完了
佐藤> ★来月からB案件にリソースを移す
鈴木> 予算は？？？（聞き取れず→後で確認）
田中> 500万の範囲内で
```

### 重要な発言のハイライト方法
| 記号 | 意味 | 使い方 |
|------|------|--------|
| ★ | 重要な発言・決定 | `佐藤> ★4月からPM変更する` |
| ？？？ | 聞き取れなかった | `田中> ？？？（後で確認）` |
| → | 結論・決定への経緯 | `→ 全員合意で承認` |
| ⚠️ | 注意・リスク | `高橋> ⚠️リソース不足の懸念あり` |

## 4. 会議後の処理

### 清書の手順（15分以内を目標）
1. 走り書きを読み返し、不明点を補完
2. 決定事項・TODO・保留事項を「まとめ」セクションに整理
3. 各TODOに **担当者・依頼者・期限** を明記
4. 合意確認の期限を設定（通常3営業日）

### 共有の手順
1. 出席者全員に共有
2. 欠席者にも共有（重要な会議の場合）
3. **合意確認の期限を明記**する
4. Teamsチャットで共有通知を送る

### 合意確認の文面テンプレート
```
【議事録共有】〇月〇日 △△会議

お疲れ様です。
本日の議事録を共有いたします。

📋 議事録：[リンク]
📅 合意確認期限：〇月〇日（X）

内容に修正がある場合は、期限までにご連絡ください。
期限までにご連絡がない場合、記載内容に合意いただいたものとさせていただきます。
```

## 5. 「言った言わない」防止のための追加Tips

### 会議中にできること
- **決定事項は会議中に読み上げる** → 「今の決定を確認します。〇〇を△△する、でよろしいですか？」
- **確認の返答を記録する** → 「田中部長：承認」
- **曖昧な表現は確認する** → 「検討する」→「いつまでに、誰が検討しますか？」

### 会議後にできること
- **みなし承認の仕組みを活用** → 期限内に異議がなければ合意済み
- **議事録の修正履歴を残す** → いつ、誰が、何を修正したか
- **重要な決定はメールでも確認** → 二重の証跡

### やってはいけないこと
- ❌ 議事録を1週間以上放置する
- ❌ 発言者を記録しない
- ❌ 「大体合意」「概ねOK」など曖昧な表現で終わらせる
- ❌ 決定事項と意見を混在させる
- ❌ 一人だけの視点で書く（客観性を欠く）

---

## 6. テンプレートのカスタマイズ

### 議題の追加方法（走り書き版）
```
---

## X. [新しい議題名]
報告者/発表者：

名前> 
名前> 

---
```

### 定例会議の場合
- 前回議事録のTODOを自動的に転記するルーチンを作る
- 議題は毎回同じ構成にする（報告→質疑→決定の流れ）
- ファイル名は `YYYY-MM-DD_会議名.txt` で統一

## 7. FAQ

**Q: 議事録が長くなりすぎる**
→ 発言を一字一句書かない。要点のみ記録する。

**Q: 発言者がわからなくなる**
→ 声を聞き分けるのが難しい場合、「〇〇さんの発言で合っていますか？」と確認する。

**Q: 決定事項があいまい**
→ 会議の最後に「今日の決定事項を確認させてください」と言って読み上げる。

**Q: 議事録を読んでくれない人がいる**
→ 合意確認の仕組み（みなし承認）を周知する。読まない＝合意した、という運用にする。

**Q: 修正を求められた場合は？**
→ 修正箇所と修正日を記録し、修正版を再共有する。元の記述は取り消し線で残すのが望ましい。
